AIZKRAUKLE 3D VR TOUR - PROJECT SETUP
=====================================

Target Platform: Meta Quest (Standalone Android VR)
Unity Version: Unity 6 (with Build Profiles)

COMPLETED STEPS
---------------

[x] Step 1: Project Setup (Completed)

    1.1 Android Build Profile
        - File → Build Profiles → Add Build Profile → Android
        - Switched to Android profile as active

    1.2 XR Plugin Management (Edit → Project Settings → XR Plug-in Management)
        - Android tab: Enabled OpenXR
        - Added Oculus Touch Controller Profile (Interaction Profiles)
        - Enabled Meta Quest Support (Feature Groups)

    1.3 Player Settings (Edit → Project Settings → Player → Android)
        - Color Space: Linear
        - Graphics API: OpenGLES3 (Auto disabled, Vulkan removed)
        - Minimum API Level: Android 10.0 (API 29)
        - Scripting Backend: IL2CPP
        - Target Architectures: ARM64 only

    1.4 XR Interaction Toolkit Samples Imported
        - Starter Assets
        - XR Device Simulator (removed from scene - conflicts with Quest tracking)

    1.5 Android Modules Installed (via Unity Hub)
        - Android SDK & NDK Tools
        - OpenJDK

[x] Step 2: XR Rig Setup (Completed)

    2.1 TourScene created (Assets/Scenes/TourScene.unity)

    2.2 XR Origin (VR) added
        - Right-click Hierarchy → XR → XR Origin (VR)
        - Position: (0, 0, 0)
        - Tracking Origin Mode: Floor

    2.3 Main Camera settings (XR Origin → Camera Offset → Main Camera)
        - Tracked Pose Driver: Rotation And Position
        - Clear Flags: Skybox (or Environment)
        - DO NOT add Skybox component to camera (causes issues)

    2.4 Walking Platform
        - Plane at (0, 0, 0), Scale (0.2, 1, 0.2) = 2x2m
        - Mesh Renderer disabled (invisible, collider only)

[x] Step 3: 360 Panorama System (Completed)

    IMPORTANT: Use Skybox approach, NOT sphere!

    3.1 Created SkyboxPanorama material (Assets/Materials/)
        - Shader: Skybox → Panoramic
        - Mapping: Latitude Longitude Layout
        - Assign 360 image to "Spherical (HDR)" texture slot
        - Adjust Rotation if view is inverted (try 180)

    3.2 Set as Scene Skybox (NOT camera component!)
        - Window → Rendering → Lighting
        - Environment tab → Skybox Material → SkyboxPanorama

    3.3 Image Quality Settings
        - Select all 360 images in Assets/GameAssets/Images/
        - Max Size: 4096 or 8192
        - Compression: None or High Quality
        - Generate Mip Maps: Unchecked
        - Click Apply

    FAILED APPROACHES (do not use):
    - Sphere with FlipNormals script - unreliable
    - Custom Panorama360 shader - UV issues
    - Skybox component on camera - causes conflicts

QUEST DEVELOPER SETUP
---------------------

1. Create developer account at developer.oculus.com
2. Enable Developer Mode via Meta phone app:
   - Menu → Devices → select Quest → Headset Settings → Developer Mode → ON
3. Restart Quest
4. Connect USB, allow debugging when prompted
5. In Unity: File → Build Profiles → select Quest in Run Device → Build and Run
